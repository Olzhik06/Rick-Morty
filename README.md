# Rick & Morty Explorer 

**Rick & Morty Explorer** — это web-приложение для поиска и просмотра информации о персонажах, эпизодах и локациях из мультсериала *Rick and Morty*.

Проект реализован с чётким разделением **frontend / backend**, где все внешние API вызываются **исключительно с серверной части**, что соответствует требованиям задания и современным best practices.

---

## Краткое описание проекта

Приложение позволяет:
- искать персонажей по имени;
- просматривать подробную информацию о персонажах;
- работать с эпизодами и локациями;
- сохранять понравившихся персонажей в избранное.

Для удобства навигации реализованы **пагинация**, **URL-state** и **детальные страницы**.

## Инструкции по установке и запуску

### Локальный запуск Backend

```bash
cd server
npm install
npm run dev
````

Проверка:

```
http://localhost:5000/health
```

### Локальный запуск Frontend

```bash
cd client
npm install
npm run dev
```

Создайте файл `client/.env`:

```env
VITE_API_BASE_URL=http://localhost:5000
```

## Описание процесса проектирования и разработки

1. Анализ требований и изучение Rick and Morty API
2. Проектирование архитектуры с разделением frontend и backend
3. Реализация backend-прокси для работы с внешним API
4. Создание интерфейса на React с использованием React Router
5. Реализация поиска, пагинации и детальных страниц
6. Добавление UX-функций (Favorites, Trending, Theme)
7. Деплой frontend и backend на облачные платформы

Основной упор был сделан на **чистую архитектуру**, **читаемый код** и **строгое соответствие требованиям задания**.

---

## Уникальные подходы и методологии

* Backend используется как прокси для всех внешних API-запросов
* Хранение состояния поиска и пагинации в URL (URL-state)
* Реализация Favorites через `localStorage`
* Trending персонажи с batch-запросами
* SPA-роутинг с корректной обработкой прямых URL (Vercel rewrites)
* Чёткое разделение ответственности между frontend и backend

---

## Компромиссы, принятые во время разработки

* Отказ от базы данных и авторизации в пользу `localStorage`
* Использование бесплатных тарифов Render и Vercel
* Отсутствие серверного рендеринга (SPA)
* Минимальный backend без бизнес-логики

Эти решения позволили быстрее реализовать полностью рабочее приложение в рамках тестового задания.

---

## Известные ошибки и ограничения

* Первый запрос к backend может быть медленным из-за cold start (Render Free Tier)
* Favorites не синхронизируются между устройствами
* Отсутствует авторизация и роли пользователей
* Нет серверной валидации пользовательских действий

Все ограничения являются **осознанными** и не мешают основному функционалу приложения.

---

## Почему был выбран данный технический стек

* **React + Vite** — современный стандарт для SPA с высокой производительностью
* **React Router** — удобная реализация клиентского роутинга
* **Express** — простой и надёжный backend для прокси-логики
* **Node.js** — естественный выбор для JavaScript-экосистемы
* **Vercel и Render** — быстрый и бесплатный деплой без сложной настройки

Выбранный стек полностью соответствует современным требованиям веб-разработки и условиям задания.
